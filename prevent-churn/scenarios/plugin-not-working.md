# Scenario: Plugin/Tool Not Working

## Common Issues
- Compatibility conflicts with other plugins or themes
- Configuration errors or misunderstood settings
- Feature not meeting expectations due to unclear documentation
- Missing a specific workflow the customer expected

## Key Solutions to Offer
- Debug the specific conflict or configuration issue
- Custom configuration session or walkthrough
- Alternative workflow or feature combination that meets their need
- Direct access to developer or senior support

## Example Opening
"I'm really sorry to hear the plugin isn't working as you expected. It's incredibly frustrating to invest time setting something up only to have it not deliver what you need. I genuinely want to make this right for you."

## Approach Pattern
1. **Understand the gap** — ask what they expected vs. what happened
2. **Make the commitment explicit** — tell them clearly and directly that you are committed to helping them get it working, whatever it takes. This must land. Don't bury it.
3. **Offer a guided session** — walk through setup for their specific use case
4. **Find a workaround** — combination of settings that achieves the same outcome
5. **Escalate if needed** — developer access or senior support for genuine feature gaps
6. **Feature request path** — if the feature doesn't exist, log it and offer early access
7. **Safety net** — if none of the above works, the refund is available

## Commitment Language to Use
The customer must leave with no doubt that help is available and that we will not give up on them. Use direct, clear language like:

- "I want to be clear: we are not going anywhere. If the plugin is not working, that is our problem to solve with you."
- "Please don't leave before giving us a chance to fix this. This is exactly what our support team is here for."
- "Getting this working for you is our responsibility, not yours alone."
- "I personally want to make sure this gets resolved. Let me take a direct look."

Never leave the customer feeling like support is a barrier they have to push through. Make it feel like a hand reaching toward them.

## Why This Approach Works
- Understanding the gap first avoids assuming the issue
- Stating the commitment explicitly removes the feeling that they're on their own
- A guided session makes saying "yes" low-effort for the customer
- Offering developer access signals that you're taking the issue seriously
- Keeping the refund visible but secondary maintains goodwill
