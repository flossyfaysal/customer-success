# Scenario: Plugin/Tool Not Working

## Common Issues
- Compatibility conflicts with other plugins or themes
- Configuration errors or misunderstood settings
- Feature not meeting expectations due to unclear documentation
- Missing a specific workflow the customer expected

## Key Solutions to Offer
- Debug the specific conflict or configuration issue
- Custom configuration session or walkthrough
- Alternative workflow or feature combination that meets their need
- Direct access to developer or senior support

## Example Opening
"I'm really sorry to hear the plugin isn't working as you expected. It's incredibly frustrating to invest time setting something up only to have it not deliver what you need. I genuinely want to make this right for you."

## Approach Pattern
1. **Understand the gap** — ask what they expected vs. what happened
2. **Offer a guided session** — walk through setup for their specific use case
3. **Find a workaround** — combination of settings that achieves the same outcome
4. **Escalate if needed** — developer access or senior support for genuine feature gaps
5. **Feature request path** — if the feature doesn't exist, log it and offer early access
6. **Safety net** — if none of the above works, the refund is available

## Why This Approach Works
- Understanding the gap first avoids assuming the issue
- A guided session makes saying "yes" low-effort for the customer
- Offering developer access signals that you're taking the issue seriously
- Keeping the refund visible but secondary maintains goodwill
