# Scenario: Backup Failure

## Common Issues
- Memory limits (PHP memory_limit)
- Timeout errors (max_execution_time)
- Large database or site size
- Server configuration restrictions

## Key Solutions to Offer
- Review backup logs to identify the exact bottleneck
- Manually create the backup on their behalf
- Adjust server settings (memory_limit, max_execution_time)
- Free migration after a successful backup is secured

## Example Opening
"I am so sorry to hear that you've run into trouble creating your backups. I know how frustrating it is to feel like you've tried everything only to have the process fail — especially when you just want the peace of mind that your data is safe."

## Approach Pattern
1. **Educate first** — explain the 2x disk space rule (a 30GB site needs 60GB+ free). This is not a plugin limitation — it's how all backup tools and hosting work
2. **Normalize** — "this is standard for any backup tool or hosting setup, not specific to our plugin"
3. **Give the solution** — increase hosting disk space or adjust server limits
4. **Offer to investigate** — if unclear, ask for website + cPanel access to check directly
5. **No-pressure refund** — "no issues at all if you want a refund, but we genuinely want to make this right"
6. **Partnership close** — "if you help me understand the setup better, we can find a resolution together"

## Why This Approach Works
- Educating shifts the frame — the problem is a hosting limitation anyone would face, not a plugin flaw
- Offering to investigate shows you're not deflecting responsibility
- Offering the refund freely (not as a last resort) removes pressure and builds trust
- Inviting the customer to collaborate turns confrontation into problem-solving
