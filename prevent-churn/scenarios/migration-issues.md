# Scenario: Migration Issues

## Common Issues
- Site not transferring correctly
- Broken links or missing content
- Database errors during transfer
- DNS propagation delays
- Post-migration design or permalink issues

## Key Solutions to Offer
- Review migration logs and error reports
- Perform manual migration with white-glove service
- Fix any post-migration issues (DNS, permalinks, images, databases)
- Extended support window after migration completes

## Example Opening
"I completely understand how stressful it is when a migration doesn't go as planned. Your website is your business, and when things aren't working correctly, it feels like everything is on the line. I'm truly sorry this hasn't been smooth for you."

## Approach Pattern (High Frustration / Multiple Failures)
1. **Take ownership personally** — not a ticket, not a queue — I'm handling this directly
2. **Identify root cause** — database encoding issues, server compatibility, DNS delays
3. **Commit to full restoration** — everything working exactly as before
4. **Offer extended safety net** — 30 days priority support post-migration
5. **Zero-risk offer** — if still not satisfied after successful migration: full refund + free service period
6. **Urgent close** — "I'm standing by right now to jump in"

## Why This Approach Works
- Taking personal ownership breaks the cycle of feeling passed around
- Naming specific root causes shows competence and reduces uncertainty
- The zero-risk offer (refund + keep service) removes every objection
- Urgency in the close shows you're treating this as a priority
