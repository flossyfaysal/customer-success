# Prompt Library Entry 001

**Input:** Customer can't create backup (likely disk space issue) — requesting refund

---

## Approach Used

1. **Educate first** — explain the 2x disk space rule (30GB site needs 60GB+ total). This isn't a plugin limitation — it's how all backup tools and hosting work
2. **Normalize** — "this is a regular procedure, not just our plugin, any hosting and backup tool follows this"
3. **Give the solution** — increase hosting disk space
4. **Offer to investigate** — if the problem is something else, ask for website + cPanel access to check from our end
5. **No-pressure refund** — "no issues at all if you want a refund, but we want to make this right"
6. **Partnership close** — "if you help me understand it better, we can find a resolution together"

## Why This Approach Works
- Educating the customer shifts the frame — the problem isn't the plugin, it's a hosting limitation anyone would face
- Offering to investigate shows you're not deflecting — you genuinely want to help
- The refund is offered freely, not as a last resort — this removes pressure and builds trust
- Asking the customer to help you understand invites collaboration instead of confrontation

## Key Pattern
Technical limitation + education + investigation offer + no-pressure refund = low-friction retention opportunity
