# Social Media Posts Skill

## Purpose

Generate one draft post per platform (LinkedIn, Facebook, X) every Wednesday, following Awesome Motive's brand and tone guidelines for each platform. Save all three drafts to a single dated file for review before manual posting.

---

## Skill structure

```
social-media-posts/
├── SKILL.md                  - This file
├── social-media-plan.md      - Topic queue (checked = done, unchecked = pending)
├── linkedin-guidelines.md    - LinkedIn tone, templates, and guardrails
├── facebook-guidelines.md    - Facebook tone, templates, and guardrails
├── x-guidelines.md           - X/Twitter tone, templates, and guardrails
└── drafts/                   - Output folder (one file per week)
    └── YYYY-MM-DD.md
```

---

## Content plan

Topics are managed in `social-media-plan.md`. Each line uses this format:

- `- [ ] Topic title` - pending, not yet written
- `- [x] Topic title` - done, already written

**Always pick the first unchecked item** (`- [ ]`) in the file. That is the topic for this week's posts across all three platforms. After saving the draft and sending the Slack notification, update `social-media-plan.md` by changing that item from `- [ ]` to `- [x]`.

If all items are checked (plan exhausted), send a Slack notification asking for new topics to be added to `social-media-plan.md` and do not generate any posts.

---

## How to run this skill

1. **Check the date** - Determine today's date
2. **Read `social-media-plan.md`** - Find the first unchecked item (`- [ ]`) - this is the topic
3. **Read all three guidelines files** before generating any post:
   - `linkedin-guidelines.md`
   - `facebook-guidelines.md`
   - `x-guidelines.md`
4. **Generate three posts** - One per platform, each following its own guidelines
5. **Apply humanizer principles** - No AI writing patterns. Natural voice. Vary sentence length. No em dashes, no rule-of-three padding, no inflated significance language
6. **Save the output** to `drafts/YYYY-MM-DD.md` using today's date
7. **Mark the topic as done** - In `social-media-plan.md`, change `- [ ]` to `- [x]` for the topic just used
8. **Send Slack notification** - POST to the webhook URL below with a summary message

## Quality checklist (apply before saving)

**All platforms:**
- [ ] No internal tool names mentioned
- [ ] No customer names or identifying details
- [ ] No unreleased features or roadmap hints
- [ ] No metrics not already public
- [ ] Sounds like a real person, not a press release
- [ ] No AI writing patterns (no "testament to", "pivotal", "underscores", "tapestry", em dashes overuse, rule of three)

**LinkedIn:**
- [ ] Follows one of the three content pillar templates
- [ ] Professional but personal tone
- [ ] No screenshots described

**Facebook:**
- [ ] 40-80 words (up to 150 for storytelling)
- [ ] Warm, community-oriented tone
- [ ] 2-5 hashtags at the end

**X:**
- [ ] Under 280 characters (single tweet) OR a thread of 2-5 tweets
- [ ] Strong hook on the first line
- [ ] 2-3 hashtags max, placed naturally

## Slack notification

After saving the draft file, send a Slack notification. Read the webhook URL from `config.md` in this folder — look for the line starting with `SLACK_WEBHOOK=` and extract the value.

Send the notification using Bash with curl. Use this exact message format:

```bash
curl -s -X POST "$SLACK_WEBHOOK" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "*Weekly Social Media Drafts Ready*\n*Date:* [DATE]\n*Pillar:* [PILLAR]\n\n*LinkedIn:*\n[LINKEDIN_POST]\n\n*Facebook:*\n[FACEBOOK_POST]\n\n*X:*\n[X_POST]\n\nReview the full draft file at: social-media-posts/drafts/[DATE].md"
  }'
```

Replace [DATE], [PILLAR], [LINKEDIN_POST], [FACEBOOK_POST], and [X_POST] with the actual generated content. Keep each post preview to the first 2-3 lines if it is long - the full version is in the file.

For multiline content in the JSON body, escape newlines as `\n` and escape any double quotes inside the post text as `\"`.

---

## Rules

- Never post or share drafts externally - output is for internal review only
- Never fabricate specific metrics, customer quotes, or product details
- Always read the relevant guidelines file before generating each platform's post
- If given specific context or a topic by the user, use it - otherwise generate a realistic example
- Output must be paste-ready: no commentary, no explanations outside the draft file
